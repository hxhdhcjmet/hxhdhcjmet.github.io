Typst介绍

什么是Typst?

Typst是一种为科学写作而生的基于标记的排版系统,他与LaTeX属于同一类工具:编写带有特定的纯文本文件,然后通过编译器将其渲染成排版精美的PDF文档。可以说,LateX能干的(写课程作业、课堂笔记、实验报告、论文、书籍、专业Slices…),Typst也都能干。

为什么选Typst?

Typst可以看作是吸收了LaTeX几十年的经验后从头开始设计的现代化排版系统，它与LaTeX的核心差异体现在以下几个方面：

语法简洁,学习曲线平缓

Typst的标记语言非常接近Markdown,源码清晰简洁易读,不像LaTeX那样充满反括号和花括号。来个简单的对比:
LaTeX版本

Typst版本

可以看出，Typst 的代码更接近自然语言，上手难度与 Markdown 相当，无需像 LaTeX 那样记忆大量命令和宏包语法.

编译速度极快

Typst 使用 Rust 编写，采用增量编译算法。文档的编译通常在毫秒级完成，而非 LaTeX 的秒级。

这意味着你可以获得近乎实时的预览反馈，极大改善了写作体验。基本实现了写什么就显示什么。

环境搭建及其简单

使用 LaTeX 通常需要安装几个 GB 的发行版（如 TeX Live），而 Typst 的环境搭建非常简单：

  • Web App：官方提供在线编辑器，无需安装，打开浏览器即可使用，并支持协作编辑。

  • 本地编译器：只需下载一个小巧的单一二进制文件即可运行，无需管理员权限。

  • 包管理：LaTeX 的宏包需要全部安装在本地，而 Typst 的包是按需从云端自动下载并缓存的，保持了本地环境的精简。

原生支持中文

LaTeX 原生对中文的支持不佳，通常需要额外配置 xeCJK 等宏包。而 Typst 原生支持中文、日文、韩文等非拉丁语言，开箱即用，无需额外配置。

有什么缺点？

生态与成熟度:

LaTeX拥有超过四十年的发展历史,宏包生态极其庞大和成熟,几乎能覆盖所有排版要求 。相比之下,Typst的第三方包和模板库还在成长期,数量和质量都有不小差距。这是Typst的核心劣势。

AI支持较差

正是由于Typst相对较新,相关的训练数据远不及LaTeX丰富,导致当前AI回答Typst相关排版、语法问题和给出建议的能力很一般(就本人使用体验而言),经常一个问题问来问去改很久才能改正,这在当前时代不可谓不是一个大缺点。

对比Typst与LaTex

对比维度 LaTeX Typst
核心理念 稳定、成熟的排版系统，基于宏（macro）扩展 现代化的排版系统，将标记、脚本和数学模式融入其中
语法风格 繁琐，大量反斜杠和花括号，命令众多 简洁，接近 Markdown，易读易写
学习曲线 陡峭，需学习大量命令和宏包用法 平缓，上手难度与 Markdown 相当
编译速度 较慢（秒级甚至分钟级），全量编译 极快（毫秒级），支持增量编译
环境搭建 复杂，需安装数 GB 的发行版（如 TeX Live） 极简，单一二进制文件（~30MB）或 Web App
中文支持 需额外配置（如 xeCJK 宏包） 原生支持，开箱即用
编程能力 宏系统，功能强大但语法原始，调试困难 内置现代脚本语言（变量、函数、闭包、循环等）
错误信息 晦涩难懂，定位问题困难 清晰直观，带有帮助性提示
模板生态 极其庞大成熟，覆盖所有学术领域和期刊 起步阶段，数量和质量仍在成长中
第三方包 宏包生态丰富，几乎无所不能 包生态较新，常用功能已基本覆盖，小众需求可能欠缺
稳定性 极高，核心系统几十年不变 快速迭代中，语法和功能未来可能有变动
AI 辅助支持 非常成熟，主流 LLM 训练数据充足，生成质量高 明显较弱，训练数据稀缺，回答质量普遍不理想
长文档性能 稳定但编译慢，需借助 \includeonly 等技巧 日常文档极快；极端超长文档（数十万行）可能卡顿，尚需优化
协作与云支持 Overleaf 等平台提供成熟协作方案 官方提供 Web App，支持在线协作，生态正在建设中
开源与许可 完全自由开源（GPL/LPPL） 编译器开源（GPL v3），云端服务部分由商业公司运营

总结

根据个人使用体验来说,Typst更适合用来写较短的如课堂笔记、课程作业,而LaTeX则适合其他更复杂的长文档如论文、长实验报告等。
